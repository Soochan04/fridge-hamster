# PRD: 냉장고 파먹기 웹 애플리케이션 - 2단계 (레시피 생성)

## 1. 개요
* **목표:** 1단계에서 얻은 식재료 목록 데이터를 기반으로 사용자가 만들 수 있는 최적의 레시피를 생성하여 추천합니다.
* **주요 기능:**
    * 텍스트 AI 모델(`arcee-ai/trinity-large-preview:free`)에 식재료 데이터 전달 및 프롬프트 요청
    * 요리 이름, 필요한 추가 재료(선택), 조리 순서를 포함한 구조화된 레시피 카드 렌더링

## 2. 사용자 시나리오
1. 사용자가 1단계에서 확정(수정 완료)한 식재료 목록을 확인하고 '레시피 추천받기' 버튼을 클릭합니다.
2. 애플리케이션은 식재료 리스트를 모아 `arcee-ai/trinity-large-preview:free` 모델을 통해 레시피 생성을 요청합니다.
3. 사용자가 기다리는 동안 요리 관련 애니메이션이나 팁을 제공하며 체감 대기 시간을 줄입니다.
4. 분석이 완료되면, 해당 식재료로 만들 수 있는 1~3개의 레시피 옵션이 화면에 카드 형태로 나타납니다.
5. 레시피 카드에는 '요리명', '난이도/소요시간(예측)', '필요 재료', '상세 조리법'이 알아보기 쉽게 포맷팅되어 있습니다.
6. 사용자는 다른 레시피를 원할 경우 '다른 레시피 보기 (재생성)'를 요청할 수 있습니다.

## 3. 기능 요구사항
### 3.1. 프론트엔드 (UI/UX)
* **레시피 카드 UI:** 보기 편한 타이포그래피와 마크다운(Markdown) 렌더링 지원.
* **선택/재생성 버튼:** 제공된 레시피 외에 다른 결과를 원할 때 다시 요청할 수 있는 기능.
* **조건 설정 (선택 사항 확장):** 사용자가 '매운맛', '다이어트', '간단한 요리' 등 원하는 요리의 스타일을 프리셋으로 선택해 추천 결과를 필터링할 수 있는 옵션.

### 3.2. 백엔드 / API 연동
* **OpenRouter 연동:** `arcee-ai/trinity-large-preview:free` 모델을 호출.
* **프롬프트 엔지니어링:** 레시피가 일관된 JSON 구조나 잘 포맷팅된 마크다운 형태로 반환되도록 구조화된 프롬프트 작성.
  * 예시 프롬프트: "다음 제공된 식재료들만을 최대한 활용해서 만들 수 있는 맛있는 요리 레시피를 1개 추천해줘. 요리명, 소요시간, 재료 목록, 조리 순서를 마크다운 형식으로 작성해줘. 식재료: [사용자 재료 목록]"

## 4. 비기능 요구사항
* **콘텐츠 안전성:** 먹을 수 없는 조합이나 위험한 조리법이 생성되지 않도록 시스템 프롬프트 상에 가드레일(안전 장치) 제약 추가.
* **응답 속도 고려:** 텍스트 생성 모델의 특성상 응답이 길어질 수 있으므로, 스트리밍(Streaming) 방식을 지원하여 생성되는 텍스트를 사용자에게 실시간으로 보여주는 UX 고려.

## 5. 성공 지표 (Metrics)
* 레시피 생성 API 호출 성공률.
* 사용자의 '레시피 재생성' 횟수 (처음 추천된 레시피의 만족도를 간접적으로 측정).
