# PRD: 냉장고 파먹기 웹 애플리케이션 - 1단계 (이미지 인식)

## 1. 개요
* **목표:** 사용자가 업로드한 냉장고 내부 사진을 분석하여 사용 가능한 식재료 목록을 텍스트로 추출합니다.
* **주요 기능:**
    * 이미지 업로드 (파일 선택 및 드래그 앤 드롭 지원)
    * OpenRouter API (`google/gemma-3-27b-it:free` 모델)를 연동한 이미지 분석
    * 인식된 식재료 목록 텍스트 기반 UI 표시

## 2. 사용자 시나리오
1. 사용자가 웹 애플리케이션의 메인 페이지에 접속합니다.
2. '냉장고 사진 업로드' 버튼을 클릭하거나 이미지를 드래그 앤 드롭하여 사진을 제공합니다.
3. 애플리케이션은 이미지를 서버 또는 클라이언트 단에서 처리하여 `google/gemma-3-27b-it:free` 모델을 통해 분석을 요청합니다.
4. 분석 중 로딩 스피너 등의 진행 상태가 시각적으로 표시됩니다.
5. 분석이 완료되면, 현재 냉장고에 있는 식재료 목록이 화면에 텍스트 형태(예: "계란, 대파, 양파, 돼지고기 등")로 나열됩니다.
6. 사용자는 잘못 인식된 재료를 삭제하거나 누락된 재료를 수동으로 추가/수정할 수 있습니다.

## 3. 기능 요구사항
### 3.1. 프론트엔드 (UI/UX)
* **이미지 업로더:** `.jpg`, `.png`, `.jpeg` 포맷 지원, 미리보기 기능 제공.
* **로딩 인디케이터:** API 응답 대기 시간 동안 사용자의 이탈을 방지하기 위한 시각적 피드백 제공.
* **결과 표시 영역:** 분석된 텍스트를 태그나 리스트 형태로 명확하게 렌더링.
* **텍스트 에디터/태그 매니저:** 인식된 재료 목록을 사용자가 직접 수정할 수 있는 인터페이스(추가, 삭제, 이름 변경).

### 3.2. 백엔드 / API 연동
* **OpenRouter 연동:** 사용자 또는 환경설정된 API 키를 사용하여 `https://openrouter.ai/api/v1/chat/completions` 엔드포인트 호출.
* **프롬프트 엔지니어링:** 모델에게 일관된 형식(예: 쉼표로 구분된 단어 리스트 형태)으로 출력하도록 지시하는 명확한 시스템/유저 프롬프트 작성.
  * 예시 프롬프트: "이 이미지에 있는 식재료들을 정확하게 식별해서, 다른 말 없이 쉼표(,)로 구분된 단어 목록으로만 출력해줘."

## 4. 비기능 요구사항
* **보안:** `.env` 파일 등을 통해 OpenRouter API 키가 클라이언트(브라우저) 코드에 직접 노출되지 않도록 설계 (서버 경유 호출 권장).
* **성능:** 이미지 크기를 적절히 리사이징 및 압축하여 업로드 및 API 전송 속도 최적화.
* **오류 처리:** 이미지 포맷 오류, API 할당량 초과, 네트워크 오류, 빈 이미지 등에 대한 명확한 에러 메시지 제공.

## 5. 성공 지표 (Metrics)
* 이미지 업로드 성공률 (95% 이상)
* API 응답 시간 및 타임아웃 발생률 (평균 5초 이내 목표)
* 사용자의 식재료 수정 개수 (인식률의 정확도를 간접적으로 측정, 적을수록 좋음)
